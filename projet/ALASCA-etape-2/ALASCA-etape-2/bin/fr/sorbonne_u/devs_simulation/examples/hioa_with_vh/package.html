This package provides a first DEVS simulation example with HIOA models using value histories and inter/extrapolation for the evaluation of model variables at time instants that do not correspond with evaluation time instants.

The organisation of the models are as follows:

  • a model instance of TicModel emits events of type TicEvent at a chosen rate (see the run parameters) towards another model instance of HIOA_Generator;
  • the model instance of HIOA_Generator reacts to the reception of a TicEvent (external transition) by triggering an immediate internal transition which increments an exported variable by 1;
  • the model variable generated of the HIOA_Generator is imported as a model variable other by a model instance of HIOA_Multiplier which is triggered at another chosen rate (see again the run parameters) to compute another model variable mine which value is obtained by multiplying the value of the variable other by a factor that is also chosen;
  • finally, a third model instance of HIOA_Consumer imports the variable mine of HIOA_Multiplier as its own variable otherCurrent and is also triggered at a chosen rate (see again the run parameters) to set the value of its internal model variable myCurrent.

This example shows how to use the fix point initialisation protocol for HIOA models as dependencies exist between models variables. It also shows how to use and the potential impact of value histories and inter/extrapolation. To see the difference, a command line argument controls whether the run uses inter/extrapolation or not. In fact, the class SimulationMain has two command line arguments: the first tells whether the run is in real time or not, the second whether it uses inter/extrapolation or not. The javadoc of SimulationMain gives the results that must be obtained in the two cases.
